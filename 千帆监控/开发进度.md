## 域名路由方案（已实现）

### 问题

`learnbox.top` zone 上有 `*/*` 通配符 HTTP Route（指向 `class-pet-api-staging`），用于 CF 优选 SaaS 流量分发。这条路由拦截了所有 `learnbox.top` 子域名的请求，包括 `xhs-notify.learnbox.top`。

### 解决方案

在灰度 Worker（class-pet-api-staging）中新增 Service Binding + 路由转发：

**修改的文件：**
- `class_pet_api/wrangler.jsonc` — staging services 新增 `{ "binding": "XHS_NOTIFY", "service": "xhs-notify" }`
- `class_pet_api/src/index.ts` — fetch handler 最前面判断 hostname === 'xhs-notify.learnbox.top'，通过 Service Binding 转发
- `class_pet_api/src/types.ts` — Bindings 类型新增 `XHS_NOTIFY?: Fetcher`

**不需要动的：**
- `*/*` 路由 — 保持不变
- Cloudflare DNS/Route 配置 — 保持不变
- xhs-notify Worker 自身代码 — 保持不变
